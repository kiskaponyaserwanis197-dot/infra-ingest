Problem

LLM summaries are not enough for research workflows.

Source materials are fragmented

Research inputs arrive as PDFs, web pages, meeting notes, audio recordings, videos, and local files. Each format usually requires a different manual path.

Outputs are hard to audit

One-off summaries often lose source context, processing settings, prompt versions, file identity, and reproducibility.

Knowledge is hard to reuse

Without indexing and metadata, useful notes become isolated files instead of a searchable local research library.

System

A small but complete ingestion stack.

InputsFiles, URLs, audio, video, HTML, PDF, Office, CSV, JSON
ExtractMarkItDown, document parser, yt-dlp, faster-whisper
NormalizeSingle raw_text interface for downstream processing
StructureOptional schema-based OpenAI-compatible LLM layer
Observemanifest, sha256, segments, backtest, research runs
RetrieveSQLite FTS search and RAG-style Q&A over local notes

Case Study

A no-key demo for public review.

./run ingest \
  -i examples/sample_report.html \
  -o ./outputs \
  --title "AI Infra 投研接入样例" \
  --material-type research_report \
  --no-llm
The demo converts a synthetic research HTML sample into a Markdown note and manifest without sending content to a remote model. It demonstrates the public path recruiters can run immediately after cloning the repository.

Engineering Quality

Built to explain engineering tradeoffs in an interview.

Local-first privacy boundary

--no-llm mode keeps extracted text local, while local Whisper handles audio/video transcription.

Observable runs

Every run records inputs, hashes, model config, prompt version, output paths, timestamps, and generated sidecars.

Clear roadmap

Next steps include an MCP server, HTTP API, batch queue, optional vector retrieval, and richer evaluation.
